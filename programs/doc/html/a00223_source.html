3 # .. module:: mcmc
4 # :synopsis: Monte Carlo procedure
5 # .. moduleauthor:: Benjamin Audren <benjamin.audren@epfl.ch>
6 #
7 # This module defines one key function, :func:`chain`, that handles the Markov
8 # chain. So far, the code uses only one chain, as no parallelization is done.
9 #
10 # The following routine is also defined in this module, which is called at
11 # every step:
12 #
13 # * :func:`get_new_position` returns a new point in the parameter space,
14 # depending on the proposal density.
15 #
16 # The :func:`chain` in turn calls several helper routines, defined in
17 # :mod:`sampler`. These are called just once:
18 #
19 # * :func:`compute_lkl() <sampler.compute_lkl>` is called at every step in the Markov chain, returning
20 # the likelihood at the current point in the parameter space.
21 # * :func:`get_covariance_matrix() <sampler.get_covariance_matrix>`
22 # * :func:`read_args_from_chain() <sampler.read_args_from_chain>`
23 # * :func:`read_args_from_bestfit() <sampler.read_args_from_bestfit>`
24 # * :func:`accept_step() <sampler.accept_step>`
25 #
26 # Their usage is described in :mod:`sampler`. On the contrary, the following
27 # routines are called at every step:
28 #
29 # The arguments of these functions will often contain **data** and/or **cosmo**.
30 # They are both initialized instances of respectively :class:`data` and the
31 # cosmological class. They will thus not be described for every function.
32 #

47 #
48 # Obtain a new position in the parameter space from the eigen values of the
49 # inverse covariance matrix, or from the Cholesky decomposition (original
50 # idea by Anthony Lewis, in `Efficient sampling of fast and slow
51 # cosmological parameters <http://arxiv.org/abs/1304.4473>`_ )
52 #
53 # The three different jumping options, decided when starting a run with the
54 # flag **-j** are **global** (by default), **sequential** and **fast** (see
55 # :mod:`parser_mp` for reference).
56 #
57 # .. warning::
58 #
59 # For running Planck data, the option **fast** is highly recommended, as
60 # it speeds up the convergence. Note that when using this option, the
61 # list of your likelihoods in your parameter file **must match** the
62 # ordering of your nuisance parameters (as always, they must come after
63 # the cosmological parameters, but they also must be ordered between
64 # likelihood, with, preferentially, the slowest likelihood to compute
65 # coming first).
66 #
67 #
68 # - **global**: varies all the parameters at the same time. Depending on the
69 # input covariance matrix, some degeneracy direction will be followed,
70 # otherwise every parameter will jump independently of each other.
71 # - **sequential**: varies every parameter sequentially. Works best when
72 # having no clue about the covariance matrix, or to understand which
73 # estimated sigma is wrong and slowing down the whole process.
74 # - **fast**: privileged method when running the Planck likelihood. Described
75 # in the aforementioned article, it separates slow (cosmological) and fast
76 # (nuisance) parameters.
77 #
78 # Parameters
79 # ----------
80 # eigv : numpy array
81 # Eigenvalues previously computed
82 # U : numpy_array
83 # Covariance matrix.
84 # k : int
85 # Number of points so far in the chain, is used to rotate through
86 # parameters
87 # Cholesky : numpy array
88 # Cholesky decomposition of the covariance matrix, and its inverse
89 # Rotation : numpy_array
90 # Not used yet

115  if data.jumping == 'global':

117  sigmas[i] = (math.sqrt(1/eigv[i]/len(vector))) * \
118  rd.gauss(0, 1)*data.jumping_factor
119  elif data.jumping == 'sequential':

122  elif data.jumping == 'fast':

130  # parameters, this block is varied, and **only this one** (note the
131  # break at the end of the if clause, it is not a continue)
132  if i < elem:
133  if index == 0:

135  Previous = 0
136  else:

138  Previous = data.block_parameters[index-1]

140  # sigma of 1. This will translate in a jump for all the
141  # parameters (as long as the Cholesky matrix is non diagonal)
142  for j in range(Range):
143  sigmas[j+Previous] = (math.sqrt(1./Range)) * \
144  rd.gauss(0, 1)*data.jumping_factor
145  break
146  else:
147  continue
148  ####################
149  # method fast+sequential
150  #sigmas[i] = rd.gauss(0,1)*data.jumping_factor
151  else:
152  print('\n\n Jumping method unknown (accepted : ')
153  print('global (default), sequential, fast)')
154 
155  # Fill in the new vector
156  if data.jumping in ['global', 'sequential']:
157  vector_new = vector + np.dot(U, sigmas)
158  else:
159  vector_new = vector + np.dot(Cholesky, sigmas)

161  # Check for boundaries problems
162  flag = 0
163  for i, elem in enumerate(parameter_names):
164  value = data.mcmc_parameters[elem]['initial']
165  if((str(value[1]) != str(-1) and value[1] is not None) and

173  # different from zero, return False

199 ##
200 #
201 # Run a Markov chain of fixed length.
202 #
203 # Main function of this module, this is the actual Markov chain procedure.
204 # After having selected a starting point in parameter space defining the
205 # first **last accepted** one, it will, for a given amount of steps :
206 #
207 # + choose randomnly a new point following the *proposal density*,
208 # + compute the cosmological *observables* through the cosmological module,
209 # + compute the value of the *likelihoods* of the desired experiments at this point,
210 # + *accept/reject* this point given its likelihood compared to the one of
211 # the last accepted one.
212 #
213 # Every time the code accepts :code:`data.write_step` number of points
214 # (quantity defined in the input parameter file), it will write the result to
215 # disk (flushing the buffer by forcing to exit the output file, and reopen it
216 # again.
217 #
218 # .. note::
219 #
220 # to use the code to set a fiducial file for certain fixed parameters,
221 # you can use two solutions. The first one is to put all input 1-sigma
222 # proposal density to zero (this method still works, but is not
223 # recommended anymore). The second one consist in using the flag "-f 0",
224 # to force a step of zero amplitude.
225 #
226 #
227 def chain(cosmo, data, command_line):
228 
229  ## Initialisation
230  loglike = 0
231 
232  # In case command_line.silent has been asked, outputs should only contain
233  # data.out. Otherwise, it will also contain sys.stdout
234  outputs = [data.out]
235  if not command_line.silent:
236  outputs.append(sys.stdout)
237 
238  # Recover the covariance matrix according to the input, if the varying set
239  # of parameters is non-zero
240  if (data.get_mcmc_parameters(['varying']) != []):
241  sigma_eig, U, C = sampler.get_covariance_matrix(cosmo, data, command_line)
242  if data.jumping_factor == 0:
243  warnings.warn(
244  "The jumping factor has been set to 0. The above covariance " +
245  "matrix will not be used.")
246 
247  # In case of a fiducial run (all parameters fixed), simply run once and
248  # print out the likelihood. This should not be used any more (one has to
249  # modify the log.param, which is never a good idea. Instead, force the code
250  # to use a jumping factor of 0 with the option "-f 0".
251  else:
252  warnings.warn(
253  "You are running with no varying parameters... I will compute " +
254  "only one point and exit")
255  data.update_cosmo_arguments() # this fills in the fixed parameters
256  loglike = sampler.compute_lkl(cosmo, data)
257  io_mp.print_vector(outputs, 1, loglike, data)
258  return 1, loglike
259 
260  # In the fast-slow method, one need the Cholesky decomposition of the
261  # covariance matrix. Return the Cholesky decomposition as a lower
262  # triangular matrix
263  Cholesky = None
264  Inverse_Cholesky = None
265  Rotation = None
266  if command_line.jumping == 'fast':
267  Cholesky = la.cholesky(C).T
268  Inverse_Cholesky = np.linalg.inv(Cholesky)
269  Rotation = np.identity(len(sigma_eig))
270 
271  # If restart wanted, pick initial value for arguments
272  if command_line.restart is not None:
273  sampler.read_args_from_chain(data, command_line.restart)
274 
275  # If restart from best fit file, read first point (overwrite settings of
276  # read_args_from_chain)
277  if command_line.bf is not None:
278  sampler.read_args_from_bestfit(data, command_line.bf)
279 
280  # Pick a position (from last accepted point if restart, from the mean value
281  # else), with a 100 tries.
282  for i in range(100):
283  if get_new_position(data, sigma_eig, U, i,
284  Cholesky, Inverse_Cholesky, Rotation) is True:
285  break
286  if i == 99:
287  raise io_mp.ConfigurationError(
288  "You should probably check your prior boundaries... because " +
289  "no valid starting position was found after 100 tries")
290 
291  # Compute the starting Likelihood
292  loglike = sampler.compute_lkl(cosmo, data)
293 
294  # Choose this step as the last accepted value
295  # (accept_step), and modify accordingly the max_loglike
296  sampler.accept_step(data)
297  max_loglike = loglike
298 
299  # If the jumping factor is 0, the likelihood associated with this point is
300  # displayed, and the code exits.
301  if data.jumping_factor == 0:
302  io_mp.print_vector(outputs, 1, loglike, data)
303  return 1, loglike
304 
305  acc, rej = 0.0, 0.0 # acceptance and rejection number count
306  N = 1 # number of time the system stayed in the current position
307 
308  # Print on screen the computed parameters
309  io_mp.print_parameters(sys.stdout, data)
310 
311  k = 1
312  # Main loop, that goes on while the maximum number of failure is not
313  # reached, and while the expected amount of steps (N) is not taken.
314  while k <= command_line.N:
315 
316  # Pick a new position ('current' flag in mcmc_parameters), and compute
317  # its likelihood. If get_new_position returns True, it means it did not
318  # encounter any boundary problem. Otherwise, just increase the
319  # multiplicity of the point and start the loop again
320  if get_new_position(
321  data, sigma_eig, U, k, Cholesky,
322  Inverse_Cholesky, Rotation) is True:
323  newloglike = sampler.compute_lkl(cosmo, data)
324  else: # reject step
325  rej += 1
326  N += 1
327  k += 1
328  continue
329 
330  # Harmless trick to avoid exponentiating large numbers. This decides
331  # whether or not the system should move.
332  if (newloglike != data.boundary_loglike):
333  if (newloglike >= loglike):
334  alpha = 1.
335  else:
336  alpha = np.exp(newloglike-loglike)
337  else:
338  alpha = -1
339 
340  if ((alpha == 1.) or (rd.uniform(0, 1) < alpha)): # accept step
341 
342  # Print out the last accepted step (WARNING: this is NOT the one we
343  # just computed ('current' flag), but really the previous one.)
344  # with its proper multiplicity (number of times the system stayed
345  # there).
346  io_mp.print_vector(outputs, N, loglike, data)
347 
348  # Report the 'current' point to the 'last_accepted'
349  sampler.accept_step(data)
350  loglike = newloglike
351  if loglike > max_loglike:
352  max_loglike = loglike
353  acc += 1.0
354  N = 1 # Reset the multiplicity
355 
356  else: # reject step
357  rej += 1.0
358  N += 1 # Increase multiplicity of last accepted point
359 
360  # Regularly (option to set in parameter file), close and reopen the
361  # buffer to force to write on file.
362  if acc % data.write_step == 0:
363  io_mp.refresh_file(data)
364  # Update the outputs list
365  outputs[0] = data.out
366  k += 1 # One iteration done
367  # END OF WHILE LOOP
368 
369  # If at this moment, the multiplicity is higher than 1, it means the
370  # current point is not yet accepted, but it also mean that we did not print
371  # out the last_accepted one yet. So we do.
372  if N > 1:
373  io_mp.print_vector(outputs, N-1, loglike, data)
374 
375  # Print out some information on the finished chain
376  rate = acc / (acc + rej)
377  sys.stdout.write('\n# {0} steps done, acceptance rate: {1}\n'.
378  format(command_line.N, rate))
379 
380  # In case the acceptance rate is too low, or too high, print a warning
381  if rate < 0.05:
382  warnings.warn("The acceptance rate is below 0.05. You might want to "
383  "set the jumping factor to a lower value than the "
384  "default (2.4), with the option `-f 1.5` for instance.")
385  elif rate > 0.6:
386  warnings.warn("The acceptance rate is above 0.6, which means you might"
387  " have difficulties exploring the entire parameter space"
388  ". Try analysing these chains, and use the output "
389  "covariance matrix to decrease the acceptance rate to a "
390  "value between 0.2 and 0.4 (roughly).")
391 
392  # For a restart, erase the starting point to keep only the new, longer
393  # chain.
394  if command_line.restart is not None:
395  os.remove(command_line.restart)
396  sys.stdout.write(' deleting starting point of the chain {0}\n'.
397  format(command_line.restart))
398 
399  return
